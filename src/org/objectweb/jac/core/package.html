Provides JAC system core classes.

The JAC generic framework can be summarized as following:

  • Wrappable JAC objects (wrappees) automatically implement the Wrappee interface during the class loading (transformation are implemented with BCEL).
  • Classes can be excluded from translation (see the $JAC_ROOT/org.objectweb.jac.prop file).
  • The wrapper classes must extend Wrapper and can provide 3 kinds of methods:
    • wrapping methods that extends existing wrappees methods by executing some code before and after
    • role methods that add methods to the wrappees
    • exception handlers that handle exceptions thrown by the program
  • At runtime, any wrappee can be wrapped and unwrapped by any wrapper
  • Wrapping a set of wrappee with a set of wrappers must be defined in well-located components: Aspect Components
  • An aspect component (instance of AspectComponent) defines pointcuts so that it can wrap the JAC object when needed. It implements a program extension called an Aspect, e.g. authentication, persistence, distribution.
  • The aspect components set registered in the system is managed in the ACManager class.
